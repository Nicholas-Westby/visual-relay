# Manually Mark Tasks as Done

Add a per-task "Mark done" action that retires a non-archived task straight into the archive **without running it** — for tasks completed outside Visual Relay that were never moved out of the queue (e.g. `windows-support`). It reuses the existing retirement/archive machinery and reloads the list so the task leaves the queue immediately.

## Current state (researched)

- `RelayTaskItem` (`src/VisualRelay.Domain/RelayTaskItem.cs`) carries `IsArchived`, `MarkdownPath`, `TaskDirectory`, `IsNested`; `StateLabel` is `"Completed"` when archived, else `"Pending"`/`"Needs review"`.
- Retirement + archive is `TaskCompletionArchive.RetireAsync(rootPath, config, taskId, task)` (`src/VisualRelay.Core/Tasks/TaskCompletionArchive.cs`, `internal static`): renames `<id>.md` → `DONE-<id>.md` and, when `config.ArchiveOnDone`, moves it under `llm-tasks/completed/[batch-N/]`. Returns `RetirementResult` (`Additions`, `Rollback`, `DestinationPath`). Today only the driver calls it, in `RelayDriver.CommitGate.cs` `ExecuteCommitStageAsync`. It is `internal`, and VisualRelay.App has no `InternalsVisibleTo` into Core — so the App cannot call it directly.
- `RelayTaskRepository` (`src/VisualRelay.Core/Tasks/RelayTaskRepository.cs`, `public`, not `partial`) lists pending via `ListAsync`/`ListPendingAsync` (skips `DONE-`/`IGNORE-` names in `Walk`/`IsSkippedName`) and archived via `ListCompletedAsync`. File is 291 lines.
- The VM never git-commits authoring mutations: `SaveEditAsync`/`RenameAsync` (`MainWindowViewModel.Authoring.cs`) write files then `await ReloadTaskListAsync(...)`; only `RelayDriver` commits. `ReloadTaskListAsync` (`MainWindowViewModel.Helpers.cs`) re-scans and rebuilds `Tasks`. `RunBusyAsync` (same file) guards on `IsBusy` and surfaces faults to `StatusText`. `FormatQueueStatus()` is the queue-status string.
- Per-selected-task action buttons live in the `TaskDetailPanel` header (`src/VisualRelay.App/Views/Controls/TaskDetailPanel.axaml`): a `StackPanel` with `Classes="primary"` `Run selected` / `Resume`. That file is **exactly 300 lines** — the guard is `FileSizeGuard.cs` `if (lines > limit)` with `DefaultLimit = 300`, so no inline addition fits.
- `MainWindowViewModel.Commands.cs` is 298 lines — near the ceiling; new commands go in a partial.
- Confirmations use `ShowConfirmationAsync` (`MainWindowViewModel.cs`: `Func<string,string,string,Task<bool>>?`), wired in `App.axaml.cs`; `null` in tests/headless skips the prompt (see `MainWindowViewModel.Rewrite.cs` `RewriteSelectedTaskAsync`). Can-execute pattern: `CanRunSelected()` = `!IsBusy && !PauseRequested && SelectedTask is not null && !SelectedTask.IsArchived && !_rewritingTaskIds.Contains(SelectedTask.Id)`.
- Control API: `ControlApi.ResolveCommand` (`src/VisualRelay.App/Services/ControlApi.cs`) maps names → commands; `IcommandNames` in `ControlApi.State.cs` (~line 97) lists them for `/state`. Disabled commands return 409.
- Tests: `TestRepository.Create()`/`WriteConfig(testCommand, logSources, archiveOnDone: true)`/`WriteTask`/`WriteNestedTask` (`tests/VisualRelay.Tests/TestDoubles.cs`); headless UI tests use `[AvaloniaFact]` + `window.GetVisualDescendants()` (e.g. `TaskDetailRemoveButtonLayoutTests.cs`).

Keep `RelayTaskRepository.cs` ≤ 300 (it is 291); if it would exceed, mark the class `partial` and move the method into `RelayTaskRepository.MarkDone.cs`.
   - Test (mirror `TaskCompletionArchiveNoBatchTests.cs`): with `archiveOnDone:true`, `MarkDoneAsync` on a pending task moves `llm-tasks/<id>/<id>.md` under `completed/`; afterward `ListPendingAsync` excludes it and `ListCompletedAsync` includes it (`IsArchived`, `StateLabel == "Completed"`).

It won't be run by Visual Relay.``, confirm "Mark done"; on confirm, `await new RelayTaskRepository(RootPath).MarkDoneAsync(SelectedTask.Task)`, then `await ReloadTaskListAsync()` and `StatusText = FormatQueueStatus()`, all inside `RunBusyAsync`. `CanMarkSelectedTaskDone()` = `SelectedTask is not null && !SelectedTask.IsArchived && !ShowArchive && !IsBusy && !_runningTaskIds.Contains(SelectedTask.Id)`.
