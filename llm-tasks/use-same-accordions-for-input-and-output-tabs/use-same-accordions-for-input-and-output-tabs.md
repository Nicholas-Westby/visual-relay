# Use Same Accordions for Input and Output Tabs

Render the Output tab's fields with the same accordion (Expander) component the Input tab already uses for its prompt sections, instead of the static bordered boxes it uses today. This both unifies the two tabs and fixes the Output defect where Text/List field values cannot be selected — the Input accordion already shows its body in a `SelectableTextBlock`, so adopting it makes Output selectable too.

## Current state (researched)

- `ActivityColumn.axaml` hosts a `TabControl`; the Input tab (index 3) is `<controls:StageInputView/>` and the Output tab (index 4) is `<controls:StageOutputView/>`.
- **Input (the preferred component)** — `StageInputView.axaml`, branch `<!-- Ready: parsed input sections in Expanders -->`: an `ItemsControl ItemsSource="{Binding StageDetail.InputSections}"` over `PromptSection`. Each item template is `Border` (`Margin="4"`, `#2A303A`, `BorderThickness="1"`, `CornerRadius="6"`) → `Expander Header="{Binding Title}" IsExpanded="{Binding CollapsedByDefault, Converter={x:Static cvt:BoolNotConverter.Instance}}" HorizontalAlignment="Stretch" HorizontalContentAlignment="Stretch" Padding="8,4"` → `Grid RowDefinitions="Auto,*"` with a Copy `Button Click="CopyPromptSectionBody_Click"` and a `SelectableTextBlock Text="{Binding Body}"` (monospace).
- `StageInputView.axaml.cs` `CopyPromptSectionBody_Click`: `if (sender is Button { DataContext: PromptSection section } && TopLevel.GetTopLevel(this)?.Clipboard is { } clipboard) await clipboard.SetValueAsync(DataFormat.Text, section.Body);` wrapped in `try/catch` (usings `Avalonia.Input`, `Avalonia.Input.Platform`, `VisualRelay.App.Services`).
- **Output (to replace)** — `StageOutputView.axaml`, branch `<!-- Ready: rendered output fields -->`: an `ItemsControl ItemsSource="{Binding StageDetail.OutputFields}"` over `OutputField`. Each item is `Border Padding="12"` → `Grid` with `TextBlock Text="{Binding Label}"` then per-kind content gated by `OutputFieldKindEqualsConverter`: Text → plain `TextBlock` (not selectable), List → nested `ItemsControl` over `Value` via `StringSplitConverter` (not selectable), Json → `SelectableTextBlock`. Only Json is selectable today. `StageOutputView.axaml.cs` is just a ctor.
- `OutputFieldParser.cs`: `public sealed record OutputField(string Label, OutputFieldKind Kind, string Value)` and `public enum OutputFieldKind { Text, List, Json }`. `Value` is a single display string for every kind — `MapArray` joins List items with `'\n'`, Json is pretty-printed — so `Value` can be rendered directly in one `SelectableTextBlock`.
- `OutputFieldKindEqualsConverter.cs` and `StringSplitConverter.cs` are referenced **only** in `StageOutputView.axaml` (grep-confirmed; not registered in `App.axaml`), so they become dead once the template is replaced and should be deleted. `BoolNotConverter` is shared with `StageInputView.axaml`/`SettingsPanel.axaml` — leave it. Keep the `OutputFieldKind` enum (still on the record and used by the parser/tests).
- Tests: `ActivityColumnTabsUiTests` is a `[Collection("Headless")]` partial using `[AvaloniaFact]`. Helpers in `ActivityColumnTabsUiTests.cs`: `AssertContainsText(root, expected)` (reads visible `TextBlock` + `SelectableTextBlock` text) and `SwitchToTabAndFindView<T>(activityColumn, tabIndex)`. `OutputTab_Ready_RendersFieldsByKind` and `OutputTab_Ready_ShowsRenderedFieldsWithoutTogglingRawJson` (`...StageRendering.cs`) assert text presence only and still pass after the change. `OutputTab_Ready_AllFieldBordersStretchToPanelWidth` / `OutputTab_Ready_LastFieldBottomIsInsideScrollExtent` (`...AccordionLayout.cs`) measure the outer item `Border` (`Margin="4"`) and still pass; their comment "Output items are Borders, not Expanders" goes stale. `BuildOutputViewModel()` there builds Text/List/Json fields.
