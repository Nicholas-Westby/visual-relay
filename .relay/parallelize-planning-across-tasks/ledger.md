{
  "summary": "Parallelize the planning stages (1–4) across tasks in Visual Relay's queue drain, reducing wall-clock from sum-of-N to max-of-one, while keeping mutating stages (5–11) serial. Three approaches range from full git worktree isolation (safest, recommended) through process-level isolation (strong boundary, heavier IPC) to lightweight in-tree shared locks + command whitelisting (simplest, relies on stage-5 cleanup for safety).",
  "options": [
    "Option A — Git worktree isolation: ephemeral per-task worktrees outside the main repo, each running the driver with LastStageToRun=4, then copying .relay/<taskId>/ back. No lock changes needed; stray shell writes land in a throwaway tree.",
    "Option B — Process-level isolation: spawn each planning task as a separate OS process via the CLI entry point. Strongest boundary (separate process heap) but requires cross-process IPC or polling for live GUI updates.",
    "Option C — Lightweight in-tree: modify ActiveTaskLock to allow shared read leases for stages 1–4, tighten command whitelists, run all planning in the same working directory. Minimal new infrastructure but relies on the stage-5 red gate (git stash) as a backstop for stray writes."
